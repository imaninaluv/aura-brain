# Writing Modes

Version: 1.0  
Status: Locked  
Last Updated: 2026-07-05

---

# Purpose

Writing Modes define the user's primary content objective before HERO begins generating content.

Writing Mode is the highest-priority instruction within the generation pipeline.

Every decision made by HERO must align with the selected Writing Mode.

If a conflict exists between HERO's writing strategy and the selected Writing Mode, the Writing Mode always takes priority.

Writing Mode represents the creator's intent.

HERO exists to fulfil that intent.

---

# Philosophy

Writing Mode is not a writing style.

It is the destination.

HERO determines the journey.

Different Writing Modes may use similar psychological techniques, but they pursue different objectives.

Selecting the correct Writing Mode ensures that every generated thread matches the creator's intended outcome.

---

# Available Writing Modes

Aura AI currently supports three Writing Modes.

- Selling
- Informative
- Engagement

---

# Default Writing Mode

The default Writing Mode is:

**Selling**

Aura AI is primarily designed for creators, entrepreneurs, and digital businesses.

Most users create content to grow a business, promote a product, or attract customers.

Selling Mode therefore becomes the default starting point.

Users may freely switch to any other Writing Mode before generation.

---

# Writing Mode Priority

Writing Mode is always prioritised over HERO.

Example:

User selects:

Informative

Even if HERO determines that a Selling structure could achieve higher engagement, HERO must continue writing as an Informative thread.

User intention always overrides AI preference.

Aura AI never changes the creator's objective.

---

# Selling Mode

## Goal

Sell the solution.

Never sell the product.

The objective is to help readers imagine a better outcome after adopting the solution.

Readers should become curious enough to discover the solution naturally.

---

## Desired Reader Response

Readers should think:

"I need this."

Instead of:

"This person is trying to sell me something."

---

## CTA Requirement

Selling Mode always includes a Call-to-Action.

If no Promotion Link is provided:

Generate a relationship-building CTA.

Examples include:

- Follow for more insights.
- Let's connect.
- Let's be mutual.
- See you in the next post.

The CTA should never feel forced.

---

If a Promotion Link is provided:

Generate a curiosity-driven CTA.

Promotion should feel like a recommendation rather than a sales pitch.

---

If Link in Bio is enabled:

Naturally reference the creator's bio.

Example:

"I left everything in my bio if you're curious."

---

If both Promotion Link and Link in Bio are provided:

Blend both naturally.

Readers should feel encouraged—not pressured—to explore further.

---

## HERO Objective

Create psychological progression that ends with a natural desire to discover the solution.

---

## DO

- Sell transformation.
- Sell outcomes.
- Build curiosity.
- Focus on reader benefits.
- Keep the CTA natural.

---

## DON'T

- Push the product.
- Pressure readers.
- Sound desperate.
- Use aggressive marketing language.
- Prioritise features over outcomes.

---

# Informative Mode

## Goal

Help readers gain a new perspective.

Readers should leave the thread feeling smarter than before.

The thread should encourage discussion naturally without forcing interaction.

---

## Desired Reader Response

Readers should think:

"I never thought about it like that."

or

"That actually makes sense."

---

## CTA Requirement

A promotional CTA is not required.

Discussion is preferred.

If appropriate, HERO may naturally encourage readers to continue the conversation.

---

## HERO Objective

Teach.

Explain.

Reframe.

Leave readers with valuable insight.

---

## DO

- Educate.
- Explain clearly.
- Challenge common beliefs.
- Encourage discussion naturally.

---

## DON'T

- Lecture readers.
- Sound academic.
- Overcomplicate explanations.
- Force engagement.

---

# Engagement Mode

## Goal

Create opinions.

The objective is conversation.

Information exists only to encourage interaction.

---

## Desired Reader Response

Readers should immediately want to comment, share an opinion, or join the discussion.

---

## CTA Requirement

Every Engagement thread must end with a question.

The question should:

- Invite discussion.
- Encourage different perspectives.
- Feel natural.
- Avoid sounding forced.

The objective is meaningful conversation rather than empty engagement.

---

## HERO Objective

Create curiosity.

Create opinions.

Invite participation.

---

## DO

- Ask open-ended questions.
- Encourage discussion.
- Invite experiences.
- Welcome different opinions.

---

## DON'T

- Force comments.
- Ragebait.
- Ask meaningless questions.
- Create controversy purely for engagement.

---

# Writing Mode Comparison

| Writing Mode | Primary Goal | Desired Reader Response | CTA |
|---------------|--------------|-------------------------|-----|
| Selling | Sell the solution | "I need this." | Always required |
| Informative | Teach a new perspective | "I learned something." | Optional |
| Engagement | Create conversation | "I want to comment." | Always ends with a question |

---

# Writing Mode Rules

Every generated thread must follow only one Writing Mode.

Aura AI never combines multiple Writing Modes within a single thread.

If users require different objectives, they should generate multiple threads separately.

This keeps every thread focused and aligned with the creator's original intention.

---

# Engine Principle

Writing Mode defines the destination.

HERO defines the journey.

AURA ensures the destination has been reached.

The creator defines the objective.

Aura AI executes it.
