# UI / UX Guidelines

Version: 1.0
Status: Locked
Last Updated: 2026-07-05

---

# Purpose

This document defines Aura AI's user experience philosophy.

It does not define visual design.

Instead, it defines how users should experience the product.

Every screen, interaction, and workflow should follow these principles regardless of future visual redesigns.

---

# Design Philosophy

Aura AI is designed to reduce thinking.

Creators should spend their time creating and engaging—not configuring software.

Every interaction should feel:

- Fast
- Clear
- Natural
- Minimal
- Predictable

The interface should never compete with the creator's attention.

---

# Platform Strategy

Aura AI is designed using a Mobile First approach.

Primary Platform:

Mobile

Desktop exists only as a responsive experience.

Every new feature should be designed for mobile before considering larger screens.

---

# Theme

Default Theme

Dark Mode

Users may switch between:

- Dark
- Light

Theme preference is remembered automatically.

---

# Navigation

Aura AI uses a five-tab bottom navigation.

Icons only.

Home

Scout

Write

Planner

Stats

The active tab should:

- Increase slightly in size
- Display the Accent colour
- Clearly indicate the current screen

Text labels are displayed only in the page header.

Bottom navigation remains icon-only to reduce visual clutter.

---

# One Primary Action Per Screen

Every screen should have one obvious primary action.

Examples:

Home

Focus:

Complete today's mission.

---

Scout

Focus:

Approve engagement.

---

Write

Focus:

Generate content.

---

Planner

Focus:

Manage content calendar.

---

Stats

Focus:

Understand performance.

Users should never wonder what action to take next.

---

# AI Handles Complexity

Users should never interact directly with:

- HERO
- AURA
- Validation
- Scoring
- Internal prompts
- AI reasoning

All AI processing remains invisible.

Users only experience:

Input

↓

Generate

↓

Result

---

# Progressive Disclosure

Only essential options should appear initially.

Advanced Settings remain collapsed by default.

Users who need additional control may expand them.

This prevents unnecessary decision fatigue.

---

# Smart Defaults

Aura AI remembers previous user preferences.

Examples include:

- Writing Mode
- Language
- Tone

Frequently used settings should automatically become the default.

Returning users should rarely need to repeat configuration.

---

# Fast Before Fancy

Speed always takes priority over visual effects.

Animations should:

- Feel smooth
- Feel responsive
- Never delay interaction

Subtle transitions are encouraged.

Decorative animations should be avoided.

---

# Loading Experience

Loading should feel intentional.

Instead of technical messages such as:

"Generating..."

Use natural language.

Examples:

Building your thread...

Planning your content...

Reviewing your draft...

Users should always understand what Aura AI is currently doing.

---

# Confirmation Behaviour

Confirmation dialogs should appear only when actions cannot easily be undone.

Examples:

- Delete Draft
- Post Now
- Logout

Actions such as:

- Save Draft
- Generate
- Refresh
- Approve

should happen immediately without unnecessary confirmation.

Scheduling a post opens the calendar and time picker directly.

---

# Empty States

Empty screens should always guide users toward the next action.

Examples:

Draft

"No drafts yet.

Generate your first thread."

---

Planner

"No scheduled posts.

Start planning your week."

---

Scout

"No new opportunities found.

Refresh Scout."

Empty states should encourage action rather than simply reporting nothing exists.

---

# Error Messages

Error messages should remain human.

Avoid technical language.

Instead of:

"Unknown Error"

Use:

"We couldn't generate your thread.

Please try again."

Users should always understand what happened and what to do next.

---

# Dashboard Philosophy

Home is not an analytics page.

Home is an action page.

The purpose of Home is answering one question:

"What should I do today?"

Analytics remain secondary.

Action remains primary.

---

# Decision Fatigue

Aura AI should reduce decisions wherever possible.

Examples:

AI decides:

- Hook strategy
- Thread structure
- Reveal count
- Psychological progression

Users decide:

- Topic
- Writing Mode
- Publish

Anything Aura AI can confidently decide should not become another user setting.

---

# Planner Philosophy

Planner functions as a Content Calendar.

Creators should instantly recognise:

- Empty days
- Busy days
- Scheduled content
- Published content
- Engagement activity

Calendar indicators should communicate activity without requiring users to open every date.

---

# Content Flow

Writing follows one predictable journey.

Write

↓

Generate

↓

Preview

↓

Edit (Optional)

↓

Post

or

↓

Save Draft

or

↓

Schedule

Users should never become lost during the publishing process.

---

# Visual Simplicity

The interface should prioritise whitespace and clarity.

Avoid unnecessary:

- Buttons
- Cards
- Popups
- Text
- Decorations

Every visible element should have a purpose.

---

# Consistency

Every screen should behave consistently.

Buttons.

Navigation.

Dialogs.

Loading.

Animations.

Interactions.

Users should never relearn the application.

---

# Accessibility

Aura AI should remain comfortable for prolonged use.

Interfaces should prioritise:

- Clear contrast
- Readable typography
- Large touch targets
- Consistent spacing
- Predictable interactions

Accessibility should never be treated as an afterthought.

---

# UX Principles

Aura AI follows these core principles.

Reduce Decision Fatigue

AI Handles Complexity

One Primary Action Per Screen

Progressive Disclosure

Fast Before Fancy

Consistency Over Features

Never Interrupt User Flow

Always Guide the Next Action

---

# Engine Principle

Aura AI should never make creators think harder.

It should make creating feel easier.
